## ⚠️ Known caveats

1. **OneSignal package not installed.** The `@onesignal/onesignal-capacitor` name was wrong; install `onesignal-cordova-plugin` and update the dynamic import in `src/lib/push.ts` (already lazy-loaded with try/catch, so it no-ops cleanly today).
2. **Bundle analyzer didn’t emit reports** under Turbopack. Run with the webpack builder instead if you need the report:
   `ANALYZE=true NEXT_BUILD_LEGACY=1 next build`
   or use `source-map-explorer` against `out/_next/static`.
3. **Dynamic routes use a `view` placeholder** (`/article/view?id=…`, `/live/view?id=…`). This is the cleanest pattern under `output: export`. Internal links already point at the placeholder paths — don’t hand-write `/article/123` URLs.
4. **Cutover is intentionally NOT done.** `/mobile` (the Expo app) is still the shipping artifact. Do the rename only after TestFlight QA confirms parity.

## 📋 What you (the human) still have to do

1. **Set env vars** before `npm run dev` or before submitting builds. Copy `.env.example` → `.env.local` and fill:
   - `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` (same as parent `/mobile`)
   - `NEXT_PUBLIC_ONESIGNAL_APP_ID`
   - `NEXT_PUBLIC_ADMOB_*` (reuse the IDs from `/mobile/eas.json`)

2. **Install OneSignal Capacitor plugin:**
   ```
   cd mobile-next
   npm i onesignal-cordova-plugin
   npx cap sync
   ```
   Then update `src/lib/push.ts` if the import path changes.

3. **Update native config:**
   - `ios/App/App/Info.plist`: add `UIBackgroundModes → audio`, `CFBundleURLTypes → sportsmockery`, `NSUserTrackingUsageDescription`, AdMob `GADApplicationIdentifier`.
   - `android/app/src/main/AndroidManifest.xml`: add `sportsmockery://` deep link `<intent-filter>`, `FOREGROUND_SERVICE_MEDIA_PLAYBACK`, AdMob `<meta-data>`.
   - Copy app icons + splash from `/mobile/assets/` (or run `npx capacitor-assets generate`).

4. **Open in Xcode and build to a device:**
   ```
   npx cap open ios
   ```
   Real-device test the OAuth deep-link flow, AdMob banner, audio background playback, and Liquid Glass blur on iPhone 12 / iOS 15+.

5. **Install Android SDK + Studio**, then:
   ```
   npx cap open android
   ```

6. **Submit to TestFlight + Play Console internal testing** — alongside the existing Expo binary, NOT replacing it.

7. **Cutover** (only after QA sign-off):
   ```
   git mv mobile mobile-legacy
   git mv mobile-next mobile
   # update CLAUDE.md and any doc references
   git commit -m "cutover: replace Expo /mobile with Next/Capacitor build"
   ```
   Keep the legacy Expo binary in stores 30 days post-cutover via App Store Connect phased release.
